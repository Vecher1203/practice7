# Практическая работа №7 - Git, GitHub

## Цель
1. Установить систему контроля версий Git и проверить её работу.
2. Создать локальный репозиторий и научиться фиксировать изменения (commit).
3. Научиться работать с ветками: создать новую ветку, внести изменения, выполнить слияние в основную ветку.
4. Зарегистрироваться на GitHub (или использовать сузествующий аккаунт), создать удаленный репозиторий и загрузить в него локальный проект (push).
5. Разместить на GitHub ранее созданный статический сайт с использованием GitHub Pages.

---

## Ход работы

### 1. Установка Git
- Установлен Git
- Проверка успешной установки выполнена командой:
```bash
git --version
```

### 2. Создание репозитория
- Создана папка для практической работы
- Открыт терминал в этой папке и выполнена инициализация репозитория:
```bash
git init
```
- Настроено имя и email для коммитов:
git config --global user.name 
git config --global user.email 

- Построено состояние репозитория:
```bash
git status
```

### 3. Создание отчёта и добавление в репозиторий
- В корне репозитория создан файл отчёта report.md
- Файл добавлен в индекс Git:
```bash
git add report.md
```

### 4. Фиксаиця изменений
- Выполнен первый коммист с начальной версией отчёта:
```bash
git commit -m "Add initial report for practice 7"
```
- Проверена история коммитов:
```bash
git log --oneline
```

### 5. Работа с веткой: создание новой ветки и фиксация изменений
- Создана новая ветка для доработки отчёта и выполнен переход в неё:
```bash
git checkout -b report-update
```
- В файл report.md добавлено подробное описание всех шагов выполнения практической работы
- Изменения добавлены и зафиксированы в новой ветке:
```bash
git add report.md
git commit -m "Update report: add detailed steps"
```
- Проверен список веток:
```bash
git branch
```

### 6. Слияние изменений в основную ветку
- Выполнен переход в основную ветку
```bash
git checkout main
```
- Выполнено слияние ветки с доработками
```bash
git merge report-update
```
Результат: изменения из ветки report-update перенесены в основную ветку.

### 7. Публикация репозитория на GitHub
- На GitHub созда удалённый репозиторий
- Локальный репозиторий связан с удалённым:
```bash
git remote add origin https://github.com/Vecher1203/practice7.git
```
- Репозиторий загружен на GitHub:
```bash
git push -u origin main
```
- Проверка удалённого репозитория:
```bash
git remote -v
```

### 8. Размещение статического сайта через GitHub Pages
- В репозитории размещены файлы статического сайта
- Изменения добавлены и отправлены в удалённый репохиторий
```bash
git add .
git commit -m "Add static website for GitHub Pages"
git push
```
- Включен GitHub Pages
- Получена ссылка на опубликованный сайт

### Вывод
В ходе практической работы выполнены:
- установка и проверка Git
- создание локального репозитория и фиксация изменений коммитами
- работа с веткой и выполнение слияние изменений в основную ветку
- создание удалённого репозитория на GitHub и загрузка проекта
- публикация статического сайта через GitHub Pages
